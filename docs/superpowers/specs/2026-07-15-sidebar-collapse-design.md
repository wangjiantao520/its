# 侧边栏折叠修复设计

## 背景

桌面端侧边栏使用 `collapsible="icon"` 后会缩窄为图标栏，但头部、底部等宽布局内容仍保持展开状态，超出 56px 宽度并覆盖主页面。折叠后还需要保留导航图标，点击图标可正常跳转。

## 目标

- 桌面端折叠后保留 56px 宽的图标栏。
- 折叠栏不得溢出或遮挡主页面。
- 导航图标保持居中、可点击，并继续显示当前页面状态。
- 悬停图标时显示导航名称。
- 顶部折叠按钮在展开和折叠状态下均可使用。
- 移动端现有抽屉行为保持不变。

## 方案

沿用现有 shadcn `Sidebar` 的 `collapsible="icon"` 模式，不引入第二套侧边栏或新的状态管理。

1. 为头部和底部增加折叠状态样式。折叠时收紧内边距，隐藏标题、用户信息、退出按钮和版本信息，仅保留可操作的展开按钮。
2. 导航继续使用现有 `Link`，让图标本身属于链接点击区域；为菜单按钮配置标题提示，在折叠状态下通过现有 Tooltip 显示名称。
3. 对侧边栏容器与内部区域限制横向溢出，确保任何宽内容都不会覆盖主内容。
4. 保持侧边栏间隔区与实际图标栏同宽，使主内容从图标栏右侧开始布局。

## 状态与数据流

`SidebarProvider` 继续维护 `expanded` / `collapsed` 状态。头部、底部、菜单和容器仅根据父级 `data-collapsible="icon"` 应用响应式样式，不增加独立状态。菜单点击仍由 Next.js `Link` 完成路由跳转。

## 验证标准

- 折叠状态下，侧边栏间隔区和容器宽度均为 56px。
- 侧边栏内部没有元素越过其右边界。
- 每个导航图标仍位于可点击链接中，点击后 URL 与选中状态正确变化。
- 展开后标题、用户信息和完整导航文字恢复。
- 类型检查、静态检查和生产构建通过。
- 在桌面宽度下实际操作展开、折叠和图标跳转；在移动宽度下确认抽屉行为没有回归。

## 非目标

- 不重构整个侧边栏组件库。
- 不改变导航项目、权限判断或登录逻辑。
- 不修改页面主体的业务布局。
