### 新规则总结

#### 1. 维保费率规则（按设备类型）
| 设备类型 | 年维保费率 | 说明 |
|---------|-----------|------|
| 网络设备（核心交换机） | 6% | 高端网络设备 |
| 网络设备（普通交换机） | 5-6% | 普通网络设备 |
| 服务器（计算节点） | 5% | 硬件服务器 |
| 存储设备 | 4-7% | 存储系统 |
| 软件（虚拟化/云管） | 12% | 软件类产品 |
| 安全设备 | 8-12% | 安全类产品 |
| 安防设备 | 2% | 监控类设备 |
| 商密设备 | 7% | 密码类设备 |
| 服务类 | 5% | 数据迁移等服务 |
| 永久授权 | 0% | 免维保项 |

#### 2. 报价计算公式
```
年维保费用 = 设备原值 × 年维保费率
三年维保费用 = 年维保费用 × 3
```

#### 3. SLA服务标准
| 服务类型 | 频率/触发条件 | 响应要求 |
|---------|-------------|---------|
| 定期巡检 | 内网每2周1次，外网每4周1次 | 现场全量设备健康巡检 |
| 远程监控 | 7×24小时 | 异常告警15分钟内响应 |
| 一般故障响应 | 按需触发 | 2小时内到达现场，4小时内修复 |
| 重大故障响应 | 按需触发 | 1小时内到达现场，24小时内恢复业务 |
| 月度服务报告 | 每月1次 | 次月5日前提交 |
| 季度回顾会 | 每季度1次 | 与客户主管领导面对面复盘 |
| 年度维保总结 | 每年1次 | 次年1月15日前提交 |

#### 4. 服务模式
- **不驻场服务**：定期巡检 + 故障现场响应
- **内外网隔离**：严格执行内外网操作分离制度

## 技术方案

| 维度 | 选择 | 理由 |
|------|------|------|
| 数据导入方式 | API批量导入 | 支持47条设备数据批量写入 |
| 设备分类 | 按子系统分类 | 与Excel结构一致，便于查找 |
| 费率配置 | 系统参数表 | 支持动态调整各类型设备的费率 |
| SLA配置 | 系统参数表 | 支持动态调整服务标准 |

## 功能模块

### 1. 设备定额导入
- 导入47条新设备到定额库
- 设备分类：网络系统、计算资源池、数据库节点、云管理平台、存储系统、备份系统、安全平台、安防改造、业务区改造(商密)、云管理应用专业密码资源、密码应用改造
- 字段：分类、名称、品牌、型号、单价、数量、单位、维保费率、年维保费用

### 2. 维保费率配置
- 在系统参数中添加维保费率配置
- 支持按设备类型设置默认费率
- 支持在设备清单中单独调整费率

### 3. SLA服务标准配置
- 在系统参数中添加SLA配置
- 支持配置巡检频率、响应时间、修复时间等

## 实施步骤

### 阶段一：数据导入
1. **创建设备数据导入脚本** - 将47条设备数据批量导入定额库
   - 关键文件：`src/app/api/seed-cloud-datacenter/route.ts`
   - 数据：内网33类 + 外网14类设备

### 阶段二：系统参数配置
2. **添加维保费率配置** - 在系统参数表中添加各类型设备的默认费率
   - 关键文件：`src/lib/db.ts`（添加fee_rate_config表）
   - 配置：网络设备6%、服务器5%、软件12%、安全设备8-12%等

3. **添加SLA服务标准配置** - 在系统参数表中添加SLA配置
   - 关键文件：`src/lib/db.ts`（添加sla_config表）
   - 配置：巡检频率、响应时间、修复时间等

### 阶段三：前端展示
4. **更新基础数据管理页面** - 添加维保费率和SLA配置界面
   - 关键文件：`src/app/database/page.tsx`
   - 新增标签页：维保费率配置、SLA配置

### 阶段四：验证测试
5. **验证数据导入和配置功能** - 确保数据正确导入和配置生效
   - 测试：检查定额库数据、系统参数配置
